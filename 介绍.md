# 📢 ChatAnnouncement - 高级聊天公告插件（支持点击事件）

## 📋 插件信息

| 项目 | 详情 |
|------|------|
| **插件名称** | ChatAnnouncement |
| **版本** | 1.1.0 |
| **支持版本** | Minecraft 1.21.x |
| **构建工具** | Gradle |
| **依赖** | Paper/Spigot 1.21+ |
| **可选依赖** | AuthMe（注册欢迎功能） |

## 🌟 核心特性

### 🖱️ 强大的点击事件
- **填充命令到聊天框** - 点击后聊天框自动出现命令，玩家确认后执行
- **直接执行命令** - 点击后立即执行指定命令，无需额外操作
- **打开网页链接** - 点击后打开外部网页，支持投票、官网等

### 🎨 美观的消息格式
- **MiniMessage富文本** - 支持颜色、渐变、悬停提示
- **自定义前缀** - 可配置公告前缀，支持渐变效果
- **模板系统** - 预设常用模板，方便快速发送

### 🎮 完整的指令系统
- 支持向指定玩家或全体玩家发送公告
- 支持广播公告、使用模板、重载配置等功能
- 完善的权限控制，安全可靠

### 🔐 AuthMe 集成功能
- **未注册玩家提示** - 玩家加入时自动检测注册状态，未注册玩家收到注册提示
- **注册成功公告** - 玩家注册成功后全服广播欢迎消息
- **个性化欢迎** - 点击欢迎按钮自动填充欢迎消息

## 🚀 安装方法

1. **下载插件**
   - 从下方下载链接获取 `ChatAnnouncement-1.0.0.jar`

2. **安装插件**
   - 将插件放入服务器的 `plugins` 文件夹

3. **启动服务器**
   - 服务器启动后，插件会自动生成配置文件

4. **配置插件**
   - 编辑 `plugins/ChatAnnouncement/config.yml` 文件自定义设置

## 🎯 指令说明

### 🔧 核心指令

| 指令 | 语法 | 描述 | 权限 |
|------|------|------|------|
| **发送公告** | `/announcement send <玩家\|all> <消息>` | 发送公告给指定玩家或所有人 | `announcement.send` |
| **广播公告** | `/announcement broadcast <消息>` | 广播公告给所有在线玩家 | `announcement.broadcast` |
| **使用模板** | `/announcement template <模板名> <玩家\|all>` | 使用预设模板发送公告 | `announcement.template` |
| **重载配置** | `/announcement reload` | 重新加载配置文件 | `announcement.reload` |
| **查看帮助** | `/announcement help` | 显示帮助信息 | 所有玩家 |

### 📱 快捷指令

- `/ann send <玩家\|all> <消息>`
- `/notice broadcast <消息>`

## 🖱️ 点击事件详解

本插件支持三种点击事件类型，每种都有不同的触发方式和适用场景：

---

### 📝 类型一：填充命令到聊天框（suggest_command）

**触发方式**：点击后，命令会自动填充到玩家的聊天框中，玩家需要按回车键确认执行

**适用场景**：
- 需要玩家确认后再执行的命令
- 让玩家可以修改命令参数
- 安全性要求较高的操作

**语法格式**：
```yaml
<click:suggest_command:'命令内容'><hover:show_text:'悬停提示文字'><颜色>[按钮文字]</颜色></click>
```

**示例**：
```yaml
# 点击后聊天框出现 /help 命令，玩家按回车执行
<click:suggest_command:'/help'><hover:show_text:'点击获取帮助'><gold>[帮助]</gold></click>

# 点击后聊天框出现 /register 密码，玩家可以修改密码
<click:suggest_command:'/register 你的密码'><hover:show_text:'点击注册账号'><gold>[注册]</gold></click>

# 点击后聊天框出现 /msg 玩家名，玩家可以输入消息内容
<click:suggest_command:'/msg Steve '><hover:show_text:'私聊Steve'><gold>[私聊]</gold></click>
```

---

### ⚡ 类型二：直接执行命令（run_command）

**触发方式**：点击后立即执行命令，无需玩家确认，直接生效

**适用场景**：
- 安全的、无需确认的操作
- 快速传送、查看信息等
- 提升用户体验，减少操作步骤

**语法格式**：
```yaml
<click:run_command:'命令内容'><hover:show_text:'悬停提示文字'><颜色>[按钮文字]</颜色></click>
```

**示例**：
```yaml
# 点击后立即执行 /spawn 命令，传送到主城
<click:run_command:'/spawn'><hover:show_text:'传送到主城'><gold>[传送]</gold></click>

# 点击后立即执行 /rules 命令，查看服务器规则
<click:run_command:'/rules'><hover:show_text:'查看服务器规则'><gold>[规则]</gold></click>

# 点击后立即执行 /kit starter 命令，领取新手礼包
<click:run_command:'/kit starter'><hover:show_text:'领取新手礼包'><gold>[领取礼包]</gold></click>
```

---

### 🌐 类型三：打开网页链接（open_url）

**触发方式**：点击后在浏览器中打开指定的网页链接

**适用场景**：
- 投票网站链接
- 服务器官网、论坛
- 捐赠页面、商店链接
- 外部资源下载

**语法格式**：
```yaml
<click:open_url:'网页链接'><hover:show_text:'悬停提示文字'><颜色>[按钮文字]</颜色></click>
```

**示例**：
```yaml
# 点击后打开百度
<click:open_url:'https://www.baidu.com'><hover:show_text:'点击打开百度'><gold>[打开网页]</gold></click>

# 点击后打开投票网站
<click:open_url:'https://example.com/vote'><hover:show_text:'前往投票支持服务器'><gold>[投票]</gold></click>

# 点击后打开服务器官网
<click:open_url:'https://example.com'><hover:show_text:'访问服务器官网'><gold>[官网]</gold></click>

# 点击后打开捐赠页面
<click:open_url:'https://example.com/donate'><hover:show_text:'支持服务器发展'><gold>[捐赠]</gold></click>
```

### 📊 三种点击事件对比

| 类型 | 触发方式 | 是否需要确认 | 安全性 | 适用场景 |
|------|---------|-------------|--------|---------|
| **suggest_command** | 填充到聊天框 | ✅ 需要按回车 | ⭐⭐⭐⭐⭐ | 需要玩家确认的操作 |
| **run_command** | 立即执行 | ❌ 无需确认 | ⭐⭐⭐ | 安全的快速操作 |
| **open_url** | 打开网页 | ❌ 无需确认 | ⭐⭐⭐⭐ | 外部链接跳转 |

## 📝 配置文件详解

### 📁 配置文件位置
**路径**：`plugins/ChatAnnouncement/config.yml`

### 🔧 配置文件结构
```yaml
# ================================
# ChatAnnouncement 配置文件
# ================================

# 公告前缀，会自动添加到每条公告消息前面
# 支持MiniMessage富文本格式，包括颜色、渐变等
prefix: "<gradient:#FFD700:#FFA500>[公告]</gradient> "

# 预设模板，可以通过 /announcement template <模板名> <玩家|all> 使用
templates:
  # 模板名称（自定义）
  welcome:
    # 模板内容，支持完整的MiniMessage格式
    text: "<yellow>欢迎来到服务器！<click:suggest_command:'/register'><hover:show_text:'<green>点击注册账号'><gold>[点击注册]</gold></click></yellow>"
  
  spawn:
    text: "<green>点击 <click:run_command:'/spawn'><hover:show_text:'<aqua>传送到主城'><gold>[传送]</gold></click> 返回主城</green>"
  
  help:
    text: "<white>需要帮助？<click:suggest_command:'/help'><hover:show_text:'<yellow>点击获取帮助'><gold>[帮助]</gold></click> <click:run_command:'/rules'><hover:show_text:'<yellow>查看服务器规则'><gold>[规则]</gold></click></white>"
  
  vote:
    text: "<light_purple>支持服务器！<click:open_url:'https://example.com/vote'><hover:show_text:'<green>点击前往投票'><gold>[投票]</gold></click></light_purple>"
```

---

### 📝 如何新增自定义模板

**步骤一**：打开配置文件 `plugins/ChatAnnouncement/config.yml`

**步骤二**：在 `templates:` 部分添加新模板

```yaml
templates:
  # 你的新模板
  my_template:  # 模板名称，使用时输入 /announcement template my_template all
    text: "你的消息内容"  # 模板内容
```

**步骤三**：保存配置文件

**步骤四**：在游戏中执行 `/announcement reload` 重载配置

**步骤五**：使用模板 `/announcement template my_template all`

---

### ⚠️ 配置文件注意事项

1. **YAML格式**：注意缩进，使用空格而不是Tab键
2. **引号使用**：消息内容建议用双引号包裹，避免特殊字符问题
3. **颜色代码**：使用MiniMessage格式，如 `<red>`、`<gold>` 等
4. **点击事件**：确保命令以 `/` 开头，网页链接以 `http://` 或 `https://` 开头
5. **保存后重载**：修改配置后务必执行 `/announcement reload` 重载配置

## 🔐 AuthMe 集成配置

### 功能说明

本插件支持与 AuthMe 登录插件集成，提供以下功能：

| 功能 | 说明 |
|------|------|
| **未注册玩家提示** | 玩家加入服务器时，自动检测是否已注册，未注册玩家收到注册提示 |
| **注册成功公告** | 玩家注册成功后，向全服玩家广播欢迎消息 |
| **个性化欢迎** | 其他玩家点击欢迎按钮，自动填充欢迎消息 |

### 配置项

```yaml
authme:
  # 是否启用 AuthMe 集成功能
  enabled: true
  
  # 未注册玩家加入时收到的提示消息
  # %player% - 玩家名称
  unregistered-message: "<yellow>欢迎来到服务器！请点击 <click:suggest_command:'/register 密码 确认密码'><hover:show_text:'<green>点击注册账号'><gold>[注册]</gold></click> 完成注册</yellow>"
  
  # 注册成功后全服广播的消息
  # %player% - 新注册的玩家名称
  # %sender% - 点击消息的玩家名称（每个玩家看到的都不同）
  register-broadcast: "<gradient:#FFD700:#FFA500>[欢迎]</gradient> <white>欢迎 <gold>%player%</gold> 加入梦想之都服务器！点击 <click:suggest_command:'欢迎 %player%，我是 %sender%，一起游玩吧！'><hover:show_text:'<green>点击发送欢迎消息'><gold>[欢迎]</gold></click> 欢迎新玩家！</white>"
```

### 变量说明

| 变量 | 说明 | 使用场景 |
|------|------|---------|
| `%player%` | 新注册玩家的名称 | 注册成功公告 |
| `%sender%` | 点击消息的玩家名称 | 注册成功公告（每个玩家看到的都不同） |

### 使用要求

1. **安装 AuthMe 插件** - 服务器需要安装 AuthMe 插件
2. **可选依赖** - AuthMe 是可选依赖，不安装插件也能正常使用其他功能
3. **自动检测** - 插件启动时会自动检测 AuthMe 是否存在

## 🔧 常见问题

### Q: 点击事件不生效怎么办？
**A:** 请确认服务器使用的是 Paper 或 Spigot 核心，Vanilla 服务器不支持 Adventure API。

### Q: 颜色不显示怎么办？
**A:** 请使用正确的 MiniMessage 格式，例如 `<red>红色</red>`，确保服务器版本为 1.21+。

### Q: 权限不足怎么办？
**A:** 给玩家添加对应权限，例如：`/op 玩家名` 或使用权限插件管理。

### Q: 配置文件不生效怎么办？
**A:** 修改配置后运行 `/announcement reload` 重载配置。
